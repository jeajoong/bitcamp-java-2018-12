photoBoard 게시판에는 수업 정보를 불러와 선택해 저장할 수 있다.
photoBoard 게시판에서 게시물을 눌렀을 때 에러가 난 이유 및 해결방법. 

-----photoBoardService에서----- 
[lessonService의 형태를 재정의 하고 난뒤... 0,0을 줘버렸다..]

PhotoBoardController에 detail()에서 lessonService.list(int pageNo, int pageSize) 값으로 (0,0)을 줘버렸는데 
이렇게 되면 LessonServiceImpl.list()에 pageNo에 0을 줘서 rowNo가 -값을 갖고 DB의 항목을 찾는다. ->에러가 발생한다.
또한 pageSize가 0이면 해당 되는 수업의 (정보)숫자를 얻어올 수 없다.

lessonService.list()에 0,lessonService.size()를 넘겨준다. 
(size()- DAO에게 DB에서 해당 게시판의 게시물의 갯수를 가져오기 위해 SQL문으로 작성됨.)

List<Lesson> lessons = lessonService.list(0, lessonService.size());  [-lessons는 jsp에서 출력.]



-----lessonServiceImpl에서-----

lessonServiceImpl가 pageNo로 0을 받았을 때 처리해주기!

수업게시판도 게시물을 페이지당 3개씩 출력하기위해 
params.put("rowNo", (pageNo-1) * pageSize); 를 해주었지만, 
pageNo의 값으로 0이 들어가면 결국엔 rowNo이 -값으로 저장된다.

에러가 발생하므로 
if ((pageNo -1) < 0)의 값이 0보다 작다면 
params.put("rowNo",0)으로 직접 지정해주었다. => 수업데이터 PhotoBoard에서 조회용
else
params.put("rowNo",(pageNo-1) * pageSize); => 수업데이터(DB) limit 분류용
